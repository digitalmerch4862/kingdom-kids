# Ask AI MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an `Ask AI` entry near the top of the teacher/admin sidebar that opens a dedicated page where staff can ask questions about ministry data and safely perform one confirmed write action (`add points`) in the MVP.

**Architecture:** Keep the React/Vite app responsible for navigation, chat UI, preview, and confirmation. Move AI interpretation and action selection behind a controlled endpoint, then route any confirmed mutation through existing app services instead of allowing arbitrary prompt-driven writes.

**Tech Stack:** React 18, TypeScript, React Router, Vitest, Testing Library, Supabase, Gemini, existing `db.service.ts` and `ministry.service.ts`

- [ ] **Step 3: Add the service fetch wrapper**

```ts
import type { AskAIResponse } from './ask-ai.types';

interface AskAIRequest {
  prompt: string;
  actor: { role: string; username: string };
}

export class AskAIService {
  static async ask(payload: AskAIRequest): Promise<AskAIResponse> {
    const response = await fetch('/api/ask-ai', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(payload),
    });

    if (!response.ok) {
      return normalizeAskAIResponse({
        mode: 'error',
        reply: 'Ask AI is unavailable right now.',
      });
    }

    return normalizeAskAIResponse(await response.json());
  }
}
```

## Notes

- This MVP intentionally keeps only one write action (`add_points`) to reduce blast radius.
- The `/api/ask-ai` endpoint in this plan is an interface boundary, not a promise that the current frontend-only repo already has a backend. If no backend exists at implementation time, replace it with a Supabase Edge Function while keeping the same request/response contract.
- Do not reuse the current browser-exposed Gemini pattern for this feature. The MVP should keep AI interpretation off the client for any write-capable flow.
